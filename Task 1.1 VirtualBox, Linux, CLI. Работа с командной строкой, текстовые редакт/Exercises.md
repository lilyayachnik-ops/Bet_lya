Чтобы сделать полный вывод содержимого данной папки, включая скрытые файлы, используется следующая команда:

```bash

a — отображает все файлы, включая скрытые, это те, перед именем которых стоит точка или две точки
```

Для перехода между директориями используются следующие команды:

```bash
# cd .. — переход в директорию выше
# cd ~ — переход в домашнюю директорию пользователя
# cd - — переход в предыдущую рабочую директорию
# cd /home/user_name or cd ./Documents 
```
Ещё парочка полезных команд:

```bash
# sudo passwd root — задаёт пароль пользователю root, если он у вас не был задан
# su - — переключиться на root со сменой оболочки
# su - user_name — переключиться на пользователя user_name со сменой оболочки
# su user_name — без смены оболочки

Со сменой оболочки для полноценной работы от другого пользователя.

Без смены оболочки для выполнения отдельных команд с правами другого пользователя
```

Давайте разберёмся, что значит каждая колонка:
1. **Тип файла и права доступа**
    - Первый символ — это тип файла. Отметим следующие типы:
       - —=обычный файл
       - d=директория
       - l=символическая ссылка
       - b=блочное устройство
       - c=символьное устройство
    - Остальные символы — это права доступа для user, group, others.
       - r (read) — чтение (4)
       - w (write) — запись (2)
       - x (execute) — выполнение (1)
       - — право отсутствует (0)
2. **Количество ссылок** — число жестких ссылок на файл
3. **Владелец** — пользователь-owner файла
4. **Группа** — группа-owner файла
5. **Размер** — размер файла в байтах
6. **Месяц изменения** — когда файл был изменён
7. **День изменения** — число месяца
8. **Время изменения** — время последнего изменения
9. **Имя файла**

Введём название нашего любимого мультика в mouse.txt с помощью оператора перенаправления >, используя команду `echo "Spider-man" > mouse.txt`. 

**_Важно помнить_:** `>` — перенаправляет стандартный вывод в указанный файл, если файл не существует, то создает его. Каждый раз затирает файл. `>>` — тот же оператор перенаправления, но добавляет вывод в конец файла, не затерев его. Фича для создания файла `> file_name.txt`

`ls -l /bin/user > ls_output.txt 2>&1` — перенаправление стандартного вывода и вывода ошибок, где `2>&1` — нотация, а 2 — дескриптор стандартного вывода ошибок, 1 — дескриптор стандартного вывода. 0 — дескриптор стандартного ввода. Способ является старым. Есть актуальный `ls -l /usr/bin &>> ls_output.txt`.

Найдем в директории animals файл cat.txt с использованием команды `grep` и `find`. Например, `ls -l ./animals | grep cat.txt` или `find ./animals -type f -name cat.txt` Использовали `|` для того, чтобы передать вывод одной команды на ввод другой в первом и втором случае. Стоит отметить, что везде мы использовали поиск по шаблону. Давайте ещё обратим на команду `find`:
- -type f — тип файла
- -name — его имя

Давайте поговорим про параметры команды `rm`:

- -i, --interactive — запрашивать у пользователя подтверждение перед удалением. Если нет этого параметра, то команда просто удалит существующие файлы.
- -r, --recursive — рекурсивно удалить каталоги. То есть вместе с катологом будут удалены все его подкаталоги. Это обязательный параметр при удалении каталогов.
- -f, --force — игнорировать отсутствущие файлы и не запрашивать подтверждения. Отменяет действия параметра -i.
- -v, --verbose — выводить информационные сообщения в процессе удаления.
