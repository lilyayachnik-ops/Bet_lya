1. Что такое дистрибутив? Какие дистрибутивы бывают у Linux? В чем между ними разница?
    
    - **Диструбутив** — это полноценная операционная система, включающая ядро ​​Linux, системные библиотеки, утилиты, прикладное программное обеспечение и систему управления пакетами. Он создаётся путём сборки различных программных компонентов из разных источников и их объединения в единую, целостную и удобную для пользователя вычислительную среду.
    
    - **Основные компоненты дистрибутива**:

       - _Ядро Linux_: Ядро Linux является основным компонентом операционной системы, обеспечивая низкоуровневую функциональность, абстракцию оборудования и драйверы устройств
       - _Системные библиотеки_: дистрибутивы Linux включают набор системных библиотек, таких как библиотека GNU C (glibc), которые предоставляют основные функции и интерфейсы для взаимодействия приложений с базовой операционной системой.
       - _Пользовательский интерфейс_: Дистрибутивы Linux предлагают различные пользовательские интерфейсы, включая графические пользовательские интерфейсы (GUI), такие как GNOME, KDE или Xfce, а также интерфейсы командной строки (CLI). Эти интерфейсы позволяют пользователям взаимодействовать с системой и запускать приложения.
       - _Пакеты программного обеспечения_: Дистрибутивы Linux поставляются с набором пакетов программного обеспечения, включая инструменты для повышения производительности, веб-браузеры, почтовые клиенты, медиаплееры, инструменты разработки и многое другое. Эти пакеты обычно управляются и устанавливаются через систему управления пакетами.
       - _Система управления пакетами_: Дистрибутивы Linux используют систему управления пакетами для установки, обновления и удаления пакетов программного обеспечения. Примерами популярных систем управления пакетами являются Advanced Packaging Tool (APT), используемый в дистрибутивах на базе Debian, Yellowdog Updater, Modified (YUM), используемый в дистрибутивах на базе Fedora и Red Hat, и Pacman, используемый в Arch Linux.
       - _Конфигурация и настройка_: Дистрибутивы Linux предоставляют инструменты и утилиты для настройки различных аспектов системы, таких как параметры сети, отображения, учётные записи пользователей и параметры безопасности. Пользователи могут настраивать операционную систему в соответствии со своими потребностями и предпочтениями.
       - _Поддержка и сообщество_: Дистрибутивы Linux часто поддерживаются активными сообществами и форумами поддержки, где пользователи могут обратиться за помощью, поделиться знаниями и внести свой вклад в разработку и улучшение дистрибутива. Ядро Linux: Ядро Linux является основным компонентом операционной системы, обеспечивая низкоуровневую функциональность, аппаратную абстракцию и драйверы устройств.
    
     - **Виды дистрибутивов**:
       
       - Debian-based: Ubuntu, Mint, Kali
       - Red Hat-based: RHEL, CentOS, Fedora
       - Arch-based: Arch, Manjaro
       - SUSE-based: openSUSE
       - Независимые: Slackware, Gentoo
         
     - **Отличия**:
       
        - _Менеджеры пакетов_ (apt vs yum vs pacman)
        - _Целевая аудитория_ (серверы, десктоп, embedded)
        - _Политика обновлений_ (стабильные vs rolling release)
2. Что такое интерпретатор командной строки? В чем его предназначение? Какие бывают интерпретаторы?

   - **Интерпретатор** — это интерфейс между пользователем и операционной системой, который позволяет вводить команды, обрабатывать их и взаимодействовать с ядром системы. В русскоязычной литературе Shell принято называть интерпретатором командной строки.
  
   - **Виды интерпретаторов**:
     
      - _bash_: это наиболее распространенный интерпретатор в Linux. Он используется в качестве интерпретатора по умолчанию в большинстве дистрибутивов Linux.
      - _sh_: это интерпретатор командной строки POSIX, который также распространен в Linux. Он используется в качестве совместимого интерпретатора для скриптов, написанных для других систем.
      - _zsh_: одна из современных командных оболочек UNIX, используемая непосредственно как интерактивная оболочка, либо как скриптовый интерпретатор.
      - _nologin_: при назначении ее пользователю блокируется любая попытка входа под данным пользователем, очень удобно при создании пользователей под различные сервисы, а также является плюсом в рамках безопасности.
      - _csh_: она была разработана с целью включения полезных функций программирования, таких как встроенная поддержка арифметических операций и синтаксис, схожий с языком программирования C. Кроме того, он включал историю команд, отсутствовавшую в различных типах оболочек Linux, например, в оболочке Bourne. Ещё одна важная особенность оболочки C — это псевдонимы. Полный путь к оболочке C — /bin/csh. По умолчанию используется приглашение hostname# для пользователя root и hostname% для пользователей без прав root.
      - _ksh_: помимо поддержки всего того, что поддерживается оболочкой Bourne, она предоставляет пользователям новые функции. Она обеспечивает встроенную поддержку арифметических операций, а также предлагает интерактивные функции, аналогичные функциям оболочки C. Оболочка Korn запускает скрипты, написанные для оболочки Bourne, предлагая при этом возможности манипулирования строками, массивами и функциями, аналогичные языку программирования C. Она также поддерживает скрипты, написанные для оболочки C. Более того, она быстрее большинства других оболочек Linux, включая оболочку C. Полный путь к оболочке Korn — /bin/ksh. По умолчанию используется приглашение # для пользователя root и $ для пользователей без прав root.
3. В чем разница между Ubuntu Desktop и Ubuntu Server?
   
    - Desktop: Графический интерфейс, офисные приложения, драйверы
    - Server: Минимальный набор, серверное ПО, без GUI по умолчанию

4. Что такое виртуализация?

   - Виртуализация — это  это технология, которую можно использовать для создания виртуальных представлений серверов, хранилища, сетей и других физических устройств.
